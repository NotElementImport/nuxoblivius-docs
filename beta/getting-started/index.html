Getting Started


Nuxoblivius (State Managment, Store) пакет для Vue 3, сделан для удобной работы с Сервером и некоторыми мелочами проекта.

Nuxoblivius реализует работу:
  • Работу с данными (REST)
  • Хранения данных на клиенте (Сервер, Клиент)
  • Удобства и ООП для проекта
Так же содержит components для работы в во vue файлах. В виде разных плюшек.

Setup to a Project


Установка пакета, в проект.

Nuxoblivius поддерживает Vue 3 и Nuxt 3
Качаем пакет:
npm i nuxoblivius@0.4.2-b
В Nuxt 3 надо указать дополнительный файл настроек в plugins.
import { setCustomCookie, setCustomRouter, setCustomFetch, setHeaders } from "nuxoblivius"

export default defineNuxtPlugin((nuxtApp) => {
    setCustomCookie(useCookie) // Регистрируем Cookie на сервере
    setCustomRouter(useRouter) // Регистрируем Router проекта на сервере
    setCustomFetch(useLazyFetch) // Регистрируем Кастомный Fetch проекта на сервере
    setHeaders(useRequestHeaders()) // Передаём Хэдеры
})

First Store


Создание первого стора
Все сторы можно хранить в папках (module, store, models...), для примера мы создадим простой стор где будем хранить переменную: 
// MyStore.ts
import StateManager, { state } from "nuxoblivius"

export default class MyStore extends StateManager {
    // Реактивное поле testStore
    public testStore = state<string>('test')
        .one()

    constructor() {
        super("doctors") // Имя стора
        this.manage() // Запуск создание стора
    }
}
Теперь переносим выведем данные с стора 
<script setup lang='ts'>
import MyStore from '@/store/MyStore.ts'

const myStore = new MyStore()
</script>
В шаблоне 
<template>
    {{ myStore.testStore }}
    // Или
    {{ MyStore.get('testStore') }}
</template>
