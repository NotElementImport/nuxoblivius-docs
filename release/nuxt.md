# Настройка Nuxt

Данный пакет нативно поддерживает Nuxt 3

В `nuxt.config.ts` нужно прописать два поля:
```ts
export default defineNuxtConfig({
    ...
    modules: [
        'nuxoblivius/nuxt', // Подключаем модуль
    ],

    nuxoblivius: {
        logs: true, // Включает логи SSR запросов
        rules: {
            // Для SSR можно настроить. Подстройку URL запроса
            // И только для SSR
            // '/api/post' => 'http://my-api/post'
            '/api': 'http://my-api'
        }
    },
    ...
})
```

## SSR Запросы

Запросы nuxt работают через обычный `fetch` под обалочкой `useAsyncData`.
В `dev` режиме есть дубликаты запросов, это нормально, после `build` и запуска скомпилированого проекта, дубликаты исчезнут

## Pipeline

Nuxoblivius старается сделать удобным разработку вашего приложения. Что-бы лучше понимать что происходит мы распишем Pipeline для Nuxt-а

<h4> Запуск Nuxt-а </h4>

Запускается модуль `'nuxoblivius/nuxt'` который делает дружелюбные настройки для Nuxt-а а так-же обозначает о своём присуствие

<h4> Запрос с клиента </h4>

Nuxoblivius создаёт uid подключению для логов.

<h4> Вызова record (fetching data) </h4>

Nuxoblivius делает запрос через useAsyncData.

<h4> Отправка страницы клиенту (SSR Rendering) </h4>

Nuxoblivius почищяет все данные после запроса
